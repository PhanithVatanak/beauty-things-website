# Production Deployment Guide: Beauty Things Nail Studio

This workspace contains a production-ready Next.js 15 application built for **Beauty Things** in Cambodia. Out of the box, the application runs on a local JSON file-based database (`/lib/beauty_things_db.json`) allowing immediate, fully interactive, offline-first previewing and testing in AI Studio without external credentials.

To coordinate real-time transactions and persist multi-user records in production, connect this app to **Vercel** and **Supabase**.

---

## Part 1: Supabase Database Setup

1. **Create a Supabase Project**:
   - Navigate to [Supabase](https://supabase.com/) and register or log in.
   - Click **New Project**, select your organization, specify a project name (e.g., `beauty-things-nail-salon`), set your PostgreSQL password, and choose your preferred cloud hosting region (preferably Singapore `ap-southeast-1` or East Asia for Cambodian customer latency advantages).

2. **Run Schema Migrations**:
   - In your Supabase dashboard, click on the **SQL Editor** tab from the left sidebar.
   - Open a **New Query**.
   - Copy-paste the entire contents of the `/lib/supabase-schema.sql` file located in this project root.
   - Click **Run** to execute the script. This creates all tables (`products`, `orders`, `order_history`, `custom_requests`, `store_settings`, `telegram_logs`, `reviews`), defines customized columns/enums, and indexes tables.

3. **Supabase Storage Buckets Setup** (Optional for Reference Image Upload):
   - Go to the **Storage** section in your Supabase dashboard.
   - Create two public buckets:
     1. `reference-images` (For client customized reference photos).
     2. `payment-receipts` (For checkout screenshot proofs).
   - Ensure policies allow public read and authenticated write access.

## Part 2: Telegram Bot Setup (Notifications)

1. **Create a Telegram Bot**:
   - Open Telegram and search for `@BotFather`.
   - Send `/newbot` and follow the instructions to set up your bot name (e.g., `BeautyThingsNailsBot`) and username.
   - Copy the generated **HTTP API Token** (e.g., `12345678:AAH-uN294...`).

2. **Retrieve Group/Channel Chat ID**:
   - Create a Telegram Group or Channel where you (the seller) wish to receive alerts.
   - Add your newly created bot as an Administrator.
   - Ask a bot tracker user or use a chat ID lookup bot (e.g., `@RawDataBot` or `@IDBot`) to fetch your group's Chat ID (usually starts with a negative sign like `-100192847291`).

## Part 3: Deploying on Vercel (Global Hosting)

1. **Push Code to GitHub**:
   - Initialize git in your local project folder:
     ```bash
     git init
     git add .
     git commit -m "Initialize Beauty Things workspace"
     ```
   - Create a repository on GitHub (private or public) and push your files.

2. **Import Project to Vercel**:
   - Navigate to [Vercel](https://vercel.com/) and click **Add New** > **Project**.
   - Import your GitHub repository.

3. **Secure Environment Variables Configuration**:
   - In Vercel's build portal, expand the **Environment Variables** section. Add the following parameters:

   | Key | Description | Example / Recommended Value |
   | :--- | :--- | :--- |
   | `GEMINI_API_KEY` | Autogenerated at runtime in AI Studio for AI features | *Your Google AI Studio API Secret* |
   | `APP_URL` | Self-referential secure callbacks | `https://beauty-things.vercel.app` |
   | `NEXT_PUBLIC_SUPABASE_URL` | Supabase API connection route | `https://[your-id].supabase.co` |
   | `SUPABASE_SERVICE_ROLE_KEY` | Secured backend database access token | *See API Keys section in Supabase settings* |
   | `TELEGRAM_BOT_TOKEN` | Token for dispatching notifications | `12345678:AAH-uN29482...` |
   | `TELEGRAM_CHAT_ID` | Owner group notification pipeline ID | `-10019284729` |

4. **Trigger Deployment**:
   - Click **Deploy**. Vercel will bundle the Next.js App Router standalone files. Your luxury Cambodian nail salon is now live globally!

## Part 4: Production Code Adaption (Optional)

If you wish to switch `/lib/db.ts` file storage calls to direct remote Supabase REST interfaces in production, you can replace the `DatabaseManager` definitions in `/lib/db.ts` with direct standard Supabase calls:

```typescript
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!;
const supabaseRoleKey = process.env.SUPABASE_SERVICE_ROLE_KEY!;

export const supabase = createClient(supabaseUrl, supabaseRoleKey);

// e.g. replacing getProducts():
// export async function getProducts() {
//   const { data } = await supabase.from('products').select('*');
//   return data;
// }
```

This ensures complete compatibility and smooth scaling to support millions of visits in Cambodia.
